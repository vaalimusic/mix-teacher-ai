# Техническое задание: VST-плагин “Mix Teacher”

## 1. Идея проекта

**Mix Teacher** — это VST3-плагин-анализатор для Studio One и других DAW.

Плагин ставится на дорожку как обычный insert-эффект, но **не меняет звук**. Он слушает аудио в реальном времени, показывает понятные значения громкости, динамики и частотного баланса, а также даёт простые подсказки в стиле “что не так, почему это плохо и что попробовать”.

Главная цель — помочь музыканту/новичку понять, почему дорожка выбивается из микса: слишком громкая, мутная, резкая, с плохой динамикой, с лишними пиками или конфликтующими частотами.

---

## 2. Основные задачи плагина

Плагин должен:

1. Работать как VST3 insert-эффект.
2. Пропускать аудио без изменения.
3. Анализировать сигнал в реальном времени.
4. Показывать:

   * waveform;
   * spectrum;
   * peak;
   * RMS;
   * примерный LUFS short-term;
   * crest factor;
   * частотные зоны;
   * клиппинг;
   * динамические скачки.
5. Давать простые подсказки:

   * “дорожка слишком громкая”;
   * “пики близко к клиппингу”;
   * “слишком много мути”;
   * “сибилянты/резкость”;
   * “динамика скачет”;
   * “дорожка слишком плотная”.
6. Иметь красивый, простой и немного забавный интерфейс.
7. Генерировать JSON-отчёт для дальнейшей отправки в AI/Ollama.
8. В будущем — отправлять отчёт в Ollama и получать объяснение внутри плагина.

## 4. Принцип работы

Плагин ставится на дорожку:

```text
Vocal Track → Mix Teacher → EQ → Compressor → De-esser
```

Или в конец цепочки:

```text
EQ → Compressor → De-esser → Mix Teacher
```

Плагин должен иметь режимы:

```text
PRE-analysis  — анализ сырого сигнала до обработки
POST-analysis — анализ обработанного сигнала после эффектов
```

Физически это не разные режимы внутри DAW, а рекомендация пользователю, куда поставить плагин в цепочке.

## 5. Внутренняя архитектура

### 5.1 Audio Thread

В audio thread можно делать только лёгкие операции:

```text
- audio passthrough;
- peak calculation;
- RMS calculation;
- быстрый сбор samples в ring buffer;
- простая детекция клиппинга.
```

Запрещено делать в audio thread:

```text
- HTTP-запросы;
- вызовы Ollama;
- запись файлов;
- тяжёлый FFT-анализ;
- malloc/new;
- блокировки mutex;
- долгие вычисления;
- обновление UI напрямую.
```

### 5.2 Background Thread

В отдельном потоке выполняется:

```text
- FFT-анализ;
- усреднение частотных полос;
- LUFS short-term approximation;
- crest factor;
- transient analysis;
- формирование подсказок;
- генерация JSON;
- подготовка AI-запроса.
```

### 5.3 UI Thread

UI показывает:

```text
- waveform;
- spectrum;
- meters;
- подсказки;
- статус анализа;
- кнопки управления.
```

## 7. Основные измеряемые параметры

### 7.1 Громкость

Плагин должен показывать:

```text
Peak dBFS
RMS dBFS
Short-term LUFS
Crest Factor
Clipping Count
Headroom
```

Пример отображения:

```text
Peak: -2.1 dBFS        ⚠️ High
RMS: -15.4 dBFS        ⚠️ Dense
LUFS Short: -16.2      ⚠️ Loud
Crest: 7.3 dB          ⚠️ Too flat
Headroom: 2.1 dB       ⚠️ Low
```

## 8. Частотные зоны

Плагин должен анализировать энергию по частотным диапазонам.

### 8.1 Общие диапазоны

```text
20–40 Hz       sub rumble
40–80 Hz       low foundation
80–150 Hz      low body
150–350 Hz     mud
350–800 Hz     boxiness
800 Hz–2 kHz   body / tone
2–5 kHz        presence / harshness
5–9 kHz        sibilance / sharpness
9–16 kHz       air / brightness
```

### 8.2 Для вокала

```text
80–150 Hz      proximity / low buildup
180–350 Hz     mud
500–800 Hz     boxiness
2–5 kHz        intelligibility / harshness
5–9 kHz        sibilance
10–16 kHz      air
```

### 8.3 Для кика

```text
20–40 Hz       useless rumble
40–80 Hz       fundamental
80–120 Hz      body / bass conflict
200–500 Hz     boxiness
2–5 kHz        click / attack
```

### 8.4 Для баса

```text
30–60 Hz       sub
60–120 Hz      main body
120–250 Hz     warmth
400–900 Hz     note definition
1–3 kHz        string/fret attack
```

## 9. Подсказки

Плагин должен не просто показывать цифры, а объяснять проблему простым языком.

### 9.1 Пример: слишком громкая дорожка

Условие:

```text
Peak > -3 dBFS
```

Текст:

```text
Пики слишком близко к 0 dBFS.

Почему это плохо:
У дорожки почти нет запаса по громкости. Если добавить EQ, компрессию или сатурацию, звук может начать клипповать.

Что попробовать:
Убавь input gain или clip gain на 2–5 dB.
Не решай это только фейдером — сначала сделай нормальный входной уровень.
```

### 9.2 Пример: вокал мутный

Условие:

```text
Track Type = Vocal
Energy 180–350 Hz = High
```

Текст:

```text
Вокал звучит мутно.

Почему:
Слишком много энергии в районе 180–350 Hz. Эта зона часто делает голос глухим и тяжёлым.

Что попробовать:
Поставь EQ и попробуй убрать 2–4 dB в районе 220–300 Hz.
Используй Q примерно 1–2.
```

### 9.3 Пример: сибилянты

Условие:

```text
Track Type = Vocal
Spikes 5–9 kHz = High
```

Текст:

```text
Есть резкие “с” и “ш”.

Почему:
В диапазоне 5–9 kHz слишком сильные всплески. Они могут резать ухо, особенно на громкости.

Что попробовать:
Используй de-esser в районе 6–8 kHz.
Цель — уменьшать только резкие “с/ш” на 2–5 dB, а не давить весь вокал.
```

### 9.4 Пример: динамика скачет

Условие:

```text
Short-term RMS сильно меняется во времени
```

Текст:

```text
Громкость дорожки скачет.

Почему:
Одни фразы сильно громче других. Из-за этого дорожка то вылезает, то пропадает.

Что попробовать:
Сначала выровняй clip gain вручную.
Потом используй компрессор:
Ratio: 3:1
Attack: 10–30 ms
Release: 60–120 ms
Target gain reduction: 3–6 dB
```

### 9.5 Пример: кик громкий, но плохо слышен

Условие:

```text
Track Type = Kick
Low energy high
2–5 kHz low
```

Текст:

```text
Кик громкий по низу, но плохо читается.

Почему:
У него много низа, но мало атаки. На больших колонках он есть, а на маленьких может пропадать.

Что попробовать:
Не поднимай просто громкость.
Попробуй добавить 2–4 kHz на 1–3 dB.
Если есть муть, убери 250–400 Hz на 2–4 dB.
```

## 10. Уровни объяснений

Плагин должен иметь 3 режима сложности:

```text
Beginner
Intermediate
Advanced
```

### Beginner

Пример:

```text
Дорожка слишком громкая.
Убавь её входную громкость на 3 dB.
```

### Intermediate

Пример:

```text
Peak близко к 0 dBFS, а RMS довольно высокий.
Сначала убавь input gain, потом проверь компрессию.
```

### Advanced

Пример:

```text
Peak: -2.1 dBFS, RMS: -15.4 dBFS, Crest Factor: 7.3 dB.
Сигнал плотный и имеет маленький запас по headroom.
Рекомендуется снизить input gain на 3–5 dB.
```

## 11. Интерфейс

Интерфейс должен быть простым, красивым и немного игровым.

Стиль:

```text
тёмный фон;
мягкие неоновые акценты;
большие читаемые значения;
эмодзи/иконки состояния;
минимум сложных графиков;
понятные карточки с подсказками.
```

### 11.1 Главный экран

Примерный макет:

```text
┌─────────────────────────────────────────────┐
│ MIX TEACHER                                 │
│ Track Type: Vocal        Mode: Beginner     │
├─────────────────────────────────────────────┤
│ WAVEFORM                                    │
│ [~~~~~~~~~~~~~ waveform ~~~~~~~~~~~~~~~~]   │
├─────────────────────────────────────────────┤
│ SPECTRUM                                    │
│ 80  250  800  2k  5k  10k                  │
│ ▂▃▆█▇▅▃▂                                     │
├─────────────────────────────────────────────┤
│ LEVELS                                      │
│ Peak: -2.1 dBFS      ⚠️ Too Hot             │
│ RMS:  -15.4 dBFS     ⚠️ Dense               │
│ LUFS: -16.2          ⚠️ Loud                │
├─────────────────────────────────────────────┤
│ TEACHER SAYS                                │
│ 😵 Вокал слишком близко к клиппингу.         │
│ Убавь input gain на 2–5 dB.                 │
├─────────────────────────────────────────────┤
│ [Reset] [Freeze] [Copy JSON] [Ask AI]       │

```

### 11.2 Забавный элемент

Можно добавить маленького персонажа-помощника:

```text
Mix Buddy
```

Он меняет настроение:

```text
🙂 Всё нормально
😬 Осторожно, пики высокие
😵 Клиппинг
🤔 Есть муть
🔥 Очень резко
🎚️ Убавь input gain
```

Но персонаж не должен мешать работе. Это просто визуальная подсказка.

## 12. Цветовые состояния

```text
Green  — всё нормально
Yellow — стоит проверить
Orange — проблема заметная
Red    — критическая проблема
Blue   — информационная подсказка
```

Примеры:

```text
Peak ниже -6 dBFS       → Green
Peak от -6 до -3 dBFS   → Yellow
Peak от -3 до -1 dBFS   → Orange
Peak выше -1 dBFS       → Red
Clipping detected       → Red
```

## 13. Кнопки

### Reset

Сбрасывает накопленный анализ.

### Freeze

Останавливает обновление значений на экране, чтобы пользователь мог спокойно прочитать.

### Copy JSON

Копирует текущий отчёт в буфер обмена.

### Ask AI

Отправляет отчёт в локальный AI/Ollama.

В первой версии кнопка `Ask AI` может быть отключена или работать через внешний сервер.

В будущем:

```text
Mix Teacher Plugin
→ JSON report
→ Local server
→ Ollama
→ AI explanation
→ text appears inside plugin
```

Пример prompt:

```text
You are a friendly mixing teacher.
Explain this audio analysis to a beginner.
Give safe starting suggestions.
Do not give extreme settings.
Explain why each suggestion helps.

Audio report:
{JSON}
```

AI не должен управлять аудио напрямую в первой версии.

## 16. Безопасные ограничения подсказок

Плагин не должен советовать экстремальные настройки.

Примеры ограничений:

```text
EQ cut: максимум -6 dB
EQ boost: максимум +4 dB
Compressor GR target: 2–6 dB
De-esser reduction: 2–5 dB
Input gain suggestion: обычно -2 до -6 dB
Saturation: только very light / optional
```

Плагин должен часто напоминать:

```text
Это стартовая точка, а не финальное правило.
Слушай в контексте микса.
```

## 20. Ограничения

Плагин не может:

```text
- видеть весь проект Studio One сам по себе;
- знать названия дорожек без участия пользователя;
- автоматически добавлять EQ/компрессор в Studio One;
- точно понимать жанр без указания пользователя;
- заменить слух инженера.
```

Плагин может:

```text
- анализировать аудио той дорожки, где он стоит;
- показывать значения в реальном времени;
- давать понятные подсказки;
- генерировать JSON;
- помогать учиться сведению;
- быть основой для AI Mix Assistant.
```

## 21. Критерии готовности v0.1

Версия v0.1 считается готовой, если:

```text
1. Плагин открывается в Studio One как VST3.
2. Аудио проходит через плагин без изменений.
3. Peak/RMS обновляются в реальном времени.
4. Spectrum отображает частотную активность.
5. При высоких пиках появляется подсказка.
6. При избытке 180–350 Hz на вокале появляется подсказка про муть.
7. При избытке 5–9 kHz на вокале появляется подсказка про сибилянты.
8. Работает Reset.
9. Работает Freeze.
10. Работает Copy JSON.
11. UI не тормозит playback.
12. Плагин не вызывает треск, клики и зависания.
```

## 22. Главная философия продукта

Плагин не должен говорить пользователю:

```text
“Сделай вот так, это правильно.”
```

Он должен говорить:

```text
“Похоже, здесь есть проблема.
Вот почему она может мешать.
Вот безопасный первый шаг, который стоит попробовать.”
```

Главная цель — не автоматическое сведение, а обучение:

```text
Видишь проблему → понимаешь причину → пробуешь исправить → слушаешь результат.
```
